## 1.3. Тема проекта

Создание сцепификации и разработка информационной системы для интернет-магазина музыкального оборудования.

## 1.4. Цель проекта

Веб-приложение для интернет-магазина музыкального оборудования, способное выдерживать высокие нагрузки. Приложение должно улучшить пользовательский опыт покупателей и сделать удобным работу администраторов. В конечном итоге, разработанная система должна сделать процесс покупок в интернет магазине удобным, быстрым и понятным.

1. Изучить особенности и способы реализации микросервисной архитектуры;
2. Разработать архитектуру системы с учётом всех необходимых микросервисов и прочих компонентов;
3. Создать описание всех планируемых функций в системе;
4. Определить стек технологий;
5. Разработать структуру базы данных;
6. Разработать Backend WebApi;
7. Разработать Frontend.

Современная торговля давно перешла с прилавков магазинов в интернет. Качественный сервис электронной коммерции - залог успеха малого и среднего бизнеса, предлагающего товары любых категорий. # 2. Документация

## 2.1. Подробное описание проекта

Данный проект представляет собой информационную систему "Интернет-магазин", спроектированную с использованием микросервисной архитектуры. Такой подход выбран для обеспечения горизонтального масштабирования, достижения высокой отказоустойчивости и поддержания работы приложения под высокими нагрузками.

Каждый микросервис в системе является stateless, что упрощает его масштабирование и развертывание. Каждый сервис обладает своей собственной базой данных PostgreSQL, что соответствует паттерну "Database per Service".

### Технологический стек

- Backend: ASP.NET Core
- ORM: Entity Framework Core
- Аутентификация: ASP.NET Core Identity , JWT
- СУБД: PostgreSQL
- Кэширование: Redis
- Брокер сообщений: RabbitMQ
- API Gateway: Ocelot
- Контейнеризация: Docker

### Функциональная декомпозиция (Микросервисы):

Система разделена на следующие ключевые домены, каждому из которых соответствует собственный микросервис:

__UserService (Микросервис пользователей)__

Отвечает за регистрацию, аутентификацию и авторизацию  пользователей.
Управляет данными профиля, сменой пароля (включая восстановление пароля) и изменением email.
Реализует ролевую модель (Администратор, Авторизованный покупатель и Неавторизованный пользователь).

__ProductService (Микросервис товаров):__

Управляет каталогом товаров: добавление, редактирование, удаление, чтение.
Реализует импорт товаров через API, фильтрацию, поиск и пагинацию.
Управляет категориями, брендами и акциями.
Отвечает за логику работы 

__CartService (Микросервис корзины):__

Обрабатывает все операции с корзиной пользователя: добавление , удаление , изменение количества товаров.
OrderService (Микросервис заказов): 
Отвечает за создание заказа на основе корзины , отслеживание его статуса  и управление способами оплаты и доставки.

__ReviewService (Микросервис отзывов):__

Позволяет пользователям оставлять отзывы, управлять ими и реализовывать процессы модерации отзывами.

__PaymentService (Микросервис оплаты):__

Интегрируется с внешними платежными шлюзами и обрабатывает проведение оплаты.

__NotificationService (Микросервис уведомлений):__

Отправляет пользователям уведомления о статусе заказа , акциях и участвует в процессах верификации.
